# OoAIBoard_Baseball

ローカルで動かす野球スコアボードアプリです。ほぼ全工程がCodexとClaudeで制作されています。

## 1. このアプリでできること

- ブラウザで野球のスコアボードを見る（配信ソフトのブラウザソースに重ねる用途を想定）。
- 別のブラウザ画面からスコアボードを操作する。
- 同じネットワーク上の別端末からスコアボードを見る、または操作する。
- チーム名、ロゴ、色、略称の見た目（拡大率・圧縮・中央揃え）などを保存して再利用する。
- 配信画面に載せやすいよう、背景色、スコアボードごとの表示位置・拡大率を端末ごとに調整する。

対応環境:

- アプリを起動するコンピューターは Windows / Linux を対象にする。
- ブラウザから見る端末は Windows、Linux、macOS、スマホを対象にする。

## 2. 前提要件

必要なもの:

- Node.js（LTS版）とnpm。`node` と `npm` にPATHが通っていること。
- コマンドを実行できる画面（Windowsなら PowerShell / Windows Terminal / コマンドプロンプト、Linuxならターミナル）。
- ブラウザで `http://localhost:52582` を開けること。

確認方法:

- `node --version` を実行して、Node.jsのバージョンが表示されればNode.jsは使える。
- `npm --version` を実行して、npmのバージョンが表示されればnpmは使える。
- 表示されない場合は、Node.jsの公式インストール手順を確認するか、ChatGPTなどに「WindowsでNode.jsとnpmをインストールしてPATHを通す方法」のように聞く。
- Linuxの場合も、使っているディストリビューション名を添えて「UbuntuでNode.jsとnpmをインストールする方法」のように聞く。

## 3. 起動方法

1. アプリのフォルダを開く。
2. 初回のみ、`npm install` を実行する（画面部品のダウンロード。数分かかることがある）。
3. `npm start` を実行する（画面のビルドが終わってからサーバーが起動する）。
4. 起動したコンピューターのブラウザで `http://localhost:52582` を開く。
5. 「スコアボードを見る」または「スコアボードを動かす」を選ぶ。

`npm start` が使えない場合:

- Node.jsとnpmがインストールされているか確認する。
- 初回の `npm install` を実行済みか確認する。
- 一度 `npm start` でビルドした後であれば、`npm run server`（`node src/server/index.mjs`）でも起動できる。

ポート番号:

- デフォルトのポート番号は `52582`。
- 環境変数 `PORT` で変更できる。すでに使われている場合は、`52583`, `52584` のように自動的に次の番号を試す。
- 起動したコンピューター自身から見る場合は、基本的に `http://localhost:52582` を使う。

## 4. 終了方法

1. ブラウザの画面を閉じる。
2. `npm start` または `npm run server` を実行している画面で、`Ctrl + C` を押して終了する。

注意:

- ブラウザを閉じるだけでは、アプリ本体（サーバー）は終了しない。
- サーバーを終了しない限り、同じPC上ではアプリが動き続ける。

## 5. 画面ごとの使い方

### トップページ（`/`）

- 「スコアボードを見る」から、配信や確認用の表示画面（Viewer Page）を開く。
- 「スコアボードを動かす」から、スコア入力やスコアボード管理（Control List Page）を行う。
- 左上の「メニュー」から、見る/動かす/設定の各画面へ移動できる。

### スコアボードを見る（Viewer Page, `/viewer`）

- 稼働中のスコアボードをすべて表示する。
- スコアボードの中央をドラッグすると表示位置を、端や角をドラッグすると拡大率を変更できる。
- 上部ツールバーで、背景色、位置対象のスコアボード、拡大率、表示位置、表示/非表示を調整できる。拡大率と表示位置は数値入力でも変更できる。
- 表示サイズ・位置などの設定は端末（ブラウザ）ごとに保存され、他端末には影響しない。
- 表示プロパティは書き出し・読み込みができ、別ブラウザや別端末へ同じ設定を移せる。

### スコアボードを動かす（Control List Page, `/control`）

- 稼働中のスコアボード一覧を見る。一覧には視覚プレビューはなく、対戦スコアをテキストで表示する。
- 「新規スコアボード作成」で新しいスコアボードを作り、そのままスコア入力画面へ移動する。
- スコアボード名はその場で変更できる。
- 「削除」は確認ダイアログを経て実行する。

### スコア入力画面（Score Input Page, `/control/:boardId`）

- ボール、ストライク、ファウル、カウント補正（B±1/S±1）、カウントRS（ボール・ストライクのみリセット）を操作する。
- 打席結果（ホームラン、ヒット、四球、死球、凡退、空振り三振、見逃し三振、その他）を入力する。四球・死球は押し出しルールに沿ってランナーを進める。
- アウトの加減、走塁死、盗塁死、チェンジ（アウト3のときのみ）を操作する。
- 一塁、二塁、三塁のランナーをON/OFFする。
- 先攻・後攻それぞれの得点を手動で+1/-1する。
- ABS表示オプションが有効な場合、ABSカウントを操作する。
- 戻る・進むで操作を取り消し、やり直しできる（`Ctrl+Z` / `Ctrl+Shift+Z`、Macは `Command` キー）。入力欄にフォーカスがある間は反応しない。履歴を戻すと、選手名・成績・ピッチャー一覧・ABS表示/対戦選手表示もその時点へ戻る。スコアボード名とチーム設定は戻らない。
- 「スコアリセット」は確認ダイアログを経て、試合の進行状態・得点・カウント・ランナー・ABS・成績・投球数を初期状態に戻す。チーム設定・選手名は残る。
- 「編集メニュー」から、スコアボード名、チーム名、ロゴ（PNG/JPEG、256x256へ自動変換）、略称、略称の拡大率（60〜180%）・横幅圧縮（30〜120%）・中央揃え、チームカラー、文字色、ABS表示/対戦選手表示オプション、チームプリセットの読み込み・保存ができる。
- 「選手名メニュー」から、先攻/後攻タブで打者1-9番の選手名・代打チェック、打数・安打数の確認、ピッチャー一覧（名前・球数・三振数）の編集・追加・削除ができる。

### 設定画面（Settings Page, `/settings`）

- 全体設定（自動削除の有効/無効、自動削除までの時間、一時演出の表示秒数）を保存する。
- チームプリセットをカード形式で作成・編集・削除する。各カードは保存ボタンを押すまでサーバーへ送らない。
- 「プリセット並べ替え」画面（`/settings/presets/reorder`）で、ドラッグしてプリセットの表示順を変更できる。
- 「スコアボード/プリセットで使われていないロゴを削除」で、参照されなくなったロゴ画像を確認の上で削除できる。

## 6. 同じネットワーク上の別端末から使う場合

前提:

- パソコン側のポート開放など、必要なネットワーク設定は済んでいるものとする。このアプリ自体はパソコンのネットワーク設定を変更しない。

アクセス方法:

- 起動したコンピューター自身から見る場合は `http://localhost:52582` を開く。
- 別端末から見る場合は、`localhost` ではなく起動したコンピューターのIPアドレスを使う。例: IPアドレスが `192.168.1.20` なら `http://192.168.1.20:52582`。
- スマホから見る場合も、同じネットワークにつながっていれば同じ考え方でアクセスする。

## 7. 自動削除について

- 稼働中のスコアボードは、アプリを再起動しても残る。
- 初期設定では、各スコアボードが24時間アクセスされなかった場合、そのスコアボードだけを自動削除する（確認は出ない）。ほかの稼働中スコアボードは残る。
- チームプリセットや共通設定は自動削除の対象外。
- 自動削除の有効/無効や時間は、設定画面から変更できる。

## 8. 困ったときに確認すること

ブラウザで開けない:

- アプリが起動しているか確認する。
- 初回の `npm install` を実行したか確認する。
- 「ビルドファイルが見つかりません」と表示される場合は、`npm start` で起動し直す。
- 起動したコンピューター自身から見る場合は `http://localhost:52582` を、別端末から見る場合は起動したコンピューターのIPアドレスを使っているか確認する。

スコアが反映されない:

- 操作画面と閲覧画面が同じスコアボードを見ているか確認する。
- ブラウザを再読み込みする。
- アプリが起動中か確認する。

ロゴがうまく表示されない:

- PNGまたはJPEG形式か確認する。透過が必要な場合はPNGを使う。
- 編集メニューでロゴ画像を選択し、保存ボタンを押したか確認する。
- 間違えた場合は、編集メニューの「ロゴ削除」を押して保存する。
- 使われなくなったロゴファイルは、設定画面の「スコアボード/プリセットで使われていないロゴを削除」で整理できる。
