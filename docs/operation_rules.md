# Operation Rules

このファイルは、スコア入力画面、閲覧画面、管理操作で発生する状態変化を整理する。
データ構造は `data_model.md`、全体の画面構成は `scoreboard_design.md` を参照する。
画面ごとのボタン一覧は `button_list.md` を参照する。

## 1. Pitch Buttons

ボール:

- ボールカウントを増やす。
- 対戦選手表示オプションが有効な場合、ピッチャーの球数を増やす。

ストライク:

- ストライクカウントを増やす。
- 対戦選手表示オプションが有効な場合、ピッチャーの球数を増やす。

カウント補正:

- ボールカウント -1
- ボールカウント +1
- ストライクカウント -1
- ストライクカウント +1

カウント補正のルール:

- カウント補正ではピッチャーの球数を増やさない。
- カウントは下限0を下回らないようにする。
- ボールは最大4表示。
- ストライクは最大3表示。

## 2. Plate Appearance Buttons

打席結果ボタン共通:

- ホームラン、ヒット、凡退、空振り三振、見逃し三振、その他は打席結果ボタンとして扱う。
- 打席結果ボタンを押した場合、対戦選手表示オプションが有効ならピッチャーの球数を1増やす。
- 打席結果ボタンを押した場合、次の打者へ必ず進める。
- 打席結果ボタンを押した場合、ボール、ストライクをリセットする。
- 走塁死、盗塁死は打席結果ボタンではないため、ピッチャーの球数を増やさず、通常は次の打者へ進めない。

ホームラン:

- `HOME RUN` を一定時間表示する。
- ランナー数 + 1 を攻撃側得点に加算する。
- ランナーをリセットする。
- ボール、ストライクをリセットする。
- 成績にホームランを加算する。
- 次の打者へ進める。

ヒット:

- 成績にヒットを加算する。
- ボール、ストライクをリセットする。
- 次の打者へ進める。
- ランナーの進塁は自動では行わず、各塁のON/OFFボタンで手動調整する。

凡退:

- 成績に凡退を加算する。
- ボール、ストライクをリセットする。
- 次の打者へ進める。
- アウトは自動では増やさず、必要に応じてアウト操作で調整する。

空振り三振:

- `K` を一定時間表示する。
- 成績に空振り三振を加算する。
- ボール、ストライクをリセットする。
- 次の打者へ進める。
- アウトは自動では増やさない。

見逃し三振:

- 逆向きの `K` を一定時間表示する。
- 成績に見逃し三振を加算する。
- ボール、ストライクをリセットする。
- 次の打者へ進める。
- アウトは自動では増やさない。

その他:

- 四死球、打撃妨害などを想定する。
- 成績にその他を加算する。
- 打数には含めない。
- ボール、ストライクをリセットする。
- 次の打者へ進める。

## 3. Out Buttons

アウトカウント -1:

- アウトを1減らす。
- アウトは0未満にしない。

アウトカウント +1:

- アウトを1増やす。
- アウトは最大3 outs表示。

走塁死:

- 現時点ではアウトカウント +1 と同じ扱い。
- 打席結果ボタンではないため、ピッチャーの球数は増やさない。
- 通常は次の打者へ進めない。

盗塁死:

- アウトを1増やす。
- 打席結果ボタンではないため、ピッチャーの球数は増やさない。
- イニング終了時も現在のバッターを維持する。

## 4. Change Button

チェンジ:

- アウトが3のときのみ使用可能。
- 表から裏、裏から次の回の表へ進める。
- ランナー、アウト、ボール、ストライクをリセットする。

例:

- 1回表から1回裏へ
- 1回裏から2回表へ

## 5. Runner Controls

ランナーON/OFF:

- 一塁、二塁、三塁をそれぞれ独立してON/OFFできる。
- ランナー操作は手動で行う。
- ヒット時の進塁は自動では行わない。
- ホームラン時は、得点加算後にすべてのランナーをOFFにする。
- チェンジ時は、すべてのランナーをOFFにする。

## 6. Manual Score Controls

得点 +1 / -1:

- 先攻、後攻それぞれの得点を手動で増減できる。
- 得点は0未満にしない。
- ホームランの自動加点とは別に、手動補正として扱う。
- 手動得点操作も戻る、進むの対象にする。

## 7. ABS Controls

ABSカウント -1 / +1:

- ABS表示オプションが有効な場合に使う。
- 先攻、後攻それぞれのABSカウントを手動で増減できる。
- ABSカウントは0未満にしない。
- ABSカウントの初期値は2回分を想定する。
- ABSカウント操作は、ボール、ストライク、アウト、得点、投球数には影響しない。
- ABSカウント操作も戻る、進むの対象にする。

対象ボタン:

- 先攻 ABSカウント -1
- 先攻 ABSカウント +1
- 後攻 ABSカウント -1
- 後攻 ABSカウント +1

## 8. Board Management Controls

スコアボード名変更:

- スコアボード名を任意の名前へ変更できる。
- 名称変更はViewer PageとControl List Pageへリアルタイム反映する。
- 名称変更は試合状態には影響しない。
- 名称変更も戻る、進むの対象にするかは実装時に判断する。
- まずは履歴対象外でもよい。

スコアボード削除:

- 手動削除では必ず確認ダイアログを出す。
- 削除対象のスコアボード名を確認ダイアログに表示する。
- 確認後に削除した場合、そのスコアボードの試合状態、選手名、操作履歴を削除する。
- 24時間アクセスなしの自動削除では確認を取らない。

## 9. Viewer Property Controls

端末ごとの表示サイズ:

- Viewer Pageで端末ごとにスコアボードサイズを変更できる。
- 変更内容はブラウザ内に保存する。
- 他端末の表示サイズには影響しない。

表示プロパティの入出力:

- ブラウザ内に保存された表示プロパティを書き出せる。
- 書き出した表示プロパティを読み込める。
- 入出力対象は、端末ごとのスコアボードサイズ、表示位置、拡大率、背景色を想定する。

## 10. Undo and Redo

戻る:

- 直前の操作を取り消す。
- Keyboard shortcut: Ctrl+Z / Cmd+Z

進む:

- 取り消した操作をやり直す。
- Keyboard shortcut: Ctrl+Shift+Z / Cmd+Shift+Z

設計方針:

- 操作ごとに game state のスナップショットを保存する。
- 戻る、進むはスコアボード単位で独立させる。
- スコアボード削除時は履歴も削除する。

履歴対象:

- 投球操作
- 打席結果操作
- アウト操作
- チェンジ
- ランナーON/OFF
- 手動得点補正
- ABSカウント操作

履歴対象外でもよいもの:

- スコアボード名変更
- 閲覧画面の表示プロパティ変更
- 全体設定の変更

## 11. Realtime Sync

操作画面で何か変更したら、サーバーを経由して閲覧画面へ反映する。

基本の流れ:

1. Score Input Pageで操作する。
2. 操作内容をサーバーへ送る。
3. サーバーが状態を更新する。
4. サーバーが同じスコアボードを見ているViewer Pageへ通知する。
5. Viewer Pageが表示を更新する。

想定イベント:

- board created
- board deleted
- board updated
- board state changed
- viewer settings changed
- preset changed
- active clients changed

複数端末操作:

- 複数端末から同じスコアボードを操作できるようにする。
- サーバー側の状態を正とし、各端末は操作後に最新状態を受け取る。
- 同時操作が発生した場合は、サーバーが受け取った順に処理する。
- Viewer PageとScore Input Pageの両方に、最新状態をリアルタイム配信する。

## 12. Automatic Cleanup

自動削除:

- 初期設定では、24時間アプリにアクセスがない場合、すべての稼働中スコアボードを削除する。
- 自動削除の有効/無効と時間は、全体の設定メニューで変更できるようにする。
- 自動削除しても、チームプリセット、共通設定、閲覧画面設定は残す。
- アプリへのアクセスは、Viewer Page、Control List Page、Score Input Page、Settings Pageの表示、またはSSE（リアルタイム）接続で更新する。
- 24時間アクセスなしの自動削除では、削除確認を表示しない。

## 13. Overlay Timing

一時演出:

- ホームラン時は `HOME RUN` を表示する。
- 空振り三振時は `K` を表示する。
- 見逃し三振時は逆向きの `K` を表示する。
- 表示秒数は、スコアボードごとではなく全体の設定で管理する。
