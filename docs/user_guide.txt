Baseball Scoreboard User Guide

このファイルは、コーディングに詳しくない人がアプリを使うための説明書です。
現在の初期実装では、Node.jsでローカルサーバーを起動してブラウザからアクセスします。

1. このアプリでできること

- ブラウザで野球のスコアボードを見る。
- 別のブラウザ画面からスコアボードを操作する。
- 同じネットワーク上の別端末からスコアボードを見る、または操作する。
- チーム名、ロゴ、色などを保存して再利用する。
- 配信画面に載せやすいよう、背景色や表示サイズを調整する。

対応予定の環境:

- アプリを起動するコンピューターは、将来的にWindowsとLinuxの両対応を前提とする。
- ブラウザから見る端末は、Windows、Linux、macOS、スマホを対象にする。

2. 前提要件

この章は、アプリを起動する前に必要なものをまとめる場所です。
詳しいインストール手順はここには書きません。
必要になった場合は、別途ChatGPTなどに聞くか、各ソフトの公式インストール手順を確認してください。

Windowsで起動する場合:

- Node.jsがインストールされていること。
- npmが使えること。通常はNode.jsと一緒に入る。
- `node` と `npm` にPATHが通っていること。
- PowerShell、Windows Terminal、コマンドプロンプトなど、コマンドを実行できる画面が使えること。
- ブラウザで `http://localhost:52582` を開けること。
- 別端末からアクセスする場合は、Windows側のファイアウォールやポート許可が適切に設定されていること。

Linuxで起動する場合:

- Node.jsがインストールされていること。
- npmが使えること。通常はNode.jsと一緒に入る。
- `node` と `npm` にPATHが通っていること。
- ターミナルでコマンドを実行できること。
- ブラウザで `http://localhost:52582` を開けること。
- 別端末からアクセスする場合は、Linux側のファイアウォールやポート許可が適切に設定されていること。

ブラウザで見る端末:

- Windows、Linux、macOS、スマホの一般的なブラウザを対象にする。
- Chrome、Edge、Firefox、Safariなどの現行ブラウザを想定する。
- スマホから見る場合は、アプリを起動しているコンピューターと同じネットワークにつながっていること。

確認方法:

- `node --version` を実行して、Node.jsのバージョンが表示されればNode.jsは使える。
- `npm --version` を実行して、npmのバージョンが表示されればnpmは使える。
- 表示されない場合は、Node.jsの公式インストール手順を確認するか、ChatGPTなどに「WindowsでNode.jsとnpmをインストールしてPATHを通す方法」のように聞く。
- Linuxの場合も、使っているディストリビューション名を添えて「UbuntuでNode.jsとnpmをインストールする方法」のように聞く。

3. 起動方法

起動の流れ:

1. アプリのフォルダを開く。
2. 初回のみ、Node.jsが使える環境で `npm install` を実行する（画面部品のダウンロード。数分かかることがある）。
3. `npm start` を実行する（画面のビルドが終わってからサーバーが起動する）。
4. 起動したコンピューターのブラウザで `http://localhost:52582` を開く。
5. 「スコアボードを見る」または「スコアボードを動かす」を選ぶ。

`npm start` が使えない場合:

- Node.jsとnpmがインストールされているか確認する。
- 初回の `npm install` を実行済みか確認する。
- 一度 `npm start` でビルドした後であれば、`node src/server/index.mjs` でも起動できる。

ポート番号:

- デフォルトのポート番号は `52582`。
- もし追加で別のポート番号が必要な場合は、`52583`, `52584` のように続きの番号を使う予定。
- 起動したコンピューター自身から見る場合は、基本的に `http://localhost:52582` を使う。

今後、配布方法が決まったら書き換える項目:

- ダブルクリックで起動できるファイルを用意するかどうか
- Windows/Linuxそれぞれの詳しい起動手順

4. 終了方法

終了の流れ:

1. ブラウザの画面を閉じる。
2. `npm start` または `node src/server/index.mjs` を実行している画面で、終了操作を行う。
3. 通常は `Ctrl + C` で終了する。

注意:

- ブラウザを閉じるだけでは、アプリ本体が終了しない場合がある。
- サーバーを終了しない限り、同じPC上ではアプリが動き続ける。

5. 基本的な使い方

トップページ:

- 「スコアボードを見る」から、配信や確認用の表示画面を開く。
- 「スコアボードを動かす」から、スコア入力やスコアボード管理を行う。

スコアボードを見る:

- 稼働中のスコアボードを表示する。
- 背景色、表示位置、表示サイズを調整する。
- 表示サイズなどの設定は、端末ごとに保存する。
- 表示プロパティは書き出し、読み込みができる。

スコアボードを動かす:

- 稼働中のスコアボード一覧を見る。
- 新しいスコアボードを作る。
- スコアボードを選んでスコア入力画面へ入る。
- 不要なスコアボードを削除する。
- 手動削除時は、必ず確認画面が出る。

スコア入力画面:

- ボール、ストライク、アウトを操作する。
- 得点を手動で増減する。
- 一塁、二塁、三塁のランナーをON/OFFする。
- ホームラン、ヒット、凡退、三振などの打席結果を入力する。
- 戻る、進むで操作を取り消し、やり直しできる。
- スコア入力画面では、`Ctrl + Z`（Macは `Command + Z`）で戻る、`Ctrl + Shift + Z`（Macは `Command + Shift + Z`）で進むが使える。
- 編集メニューからチーム情報や表示オプションを設定する。
- 編集メニューからチームプリセットを読み込む。
- 編集メニューから現在のチーム情報をチームプリセットとして保存する。

設定画面:

- チームプリセットを作成、編集、削除する。
- プリセット並べ替え画面を開き、ドラッグでチームプリセットの順番を変更する。
- 並べ替えた順番は、設定画面の一覧やスコア入力画面のプリセット選択に反映される。
- スコアボード/プリセットで使われていないロゴを削除する。
- 自動削除の設定を変更する。
- 一時演出の表示秒数を変更する。

6. 同じネットワーク上の別端末から使う場合

前提:

- パソコン側のポート開放など、必要なネットワーク設定は済んでいるものとする。
- このアプリ自体は、パソコンのネットワーク設定を変更しない。

アクセス方法:

- 起動したコンピューター自身から見る場合は、`http://localhost:52582` を開く。
- 別端末から見る場合は、`localhost` ではなく、起動したコンピューターのIPアドレスを使う。
- 例: 起動したコンピューターのIPアドレスが `192.168.1.20` の場合、別端末では `http://192.168.1.20:52582` を開く。
- スマホから見る場合も、同じネットワークにつながっていれば同じ考え方でアクセスする。

実装後に書き換える項目:

- 別端末から開くURL。例: `http://起動PCのIPアドレス:52582`
- つながらない場合の確認手順

7. 自動削除について

- 稼働中のスコアボードは、アプリを再起動しても残る。
- ただし初期設定では、24時間アプリにアクセスがない場合、稼働中のスコアボードを自動削除する。
- チームプリセットや共通設定は自動削除の対象外。
- 自動削除の有効/無効や時間は、全体の設定画面から変更できる予定。

8. 困ったときに確認すること

ブラウザで開けない:

- アプリが起動しているか確認する。
- URLが正しいか確認する。
- 初回の `npm install` を実行したか確認する。
- 「ビルドファイルが見つかりません」と表示される場合は、`npm start` で起動し直す。
- 起動したコンピューター自身から見る場合は、`http://localhost:52582` を使っているか確認する。
- 同一ネットワーク上の別端末から見る場合は、ネットワーク設定を確認する。
- 別端末から見る場合は、`localhost` ではなく起動したコンピューターのIPアドレスを使っているか確認する。

スコアが反映されない:

- 操作画面と閲覧画面が同じスコアボードを見ているか確認する。
- ブラウザを再読み込みする。
- アプリが起動中か確認する。

ロゴがうまく表示されない:

- PNGまたはJPEG形式か確認する。
- 透過が必要な場合はPNGを使う。
- 編集メニューでロゴ画像を選択してから保存する。
- 画像は保存時に256x256へ変換される。
- 間違えた場合は、編集メニューのロゴ削除を押して保存する。
- 使われなくなったロゴファイルを整理したい場合は、設定画面の「スコアボード/プリセットで使われていないロゴを削除」を使う。

9. 実装後に必ず更新する項目

- 起動方法
- 終了方法
- トップページURL。基本は `http://localhost:52582`
- 別端末からアクセスするURL。例: `http://起動PCのIPアドレス:52582`
- 実際の画面名やボタン名
- トラブル時の具体的な確認手順
