# Data Model

このファイルは、野球スコアボードアプリで扱うデータ構造を整理する。
画面構成や実装順は `scoreboard_design.md`、操作時の状態変化は `operation.md` を参照する。

## 1. Board

稼働中のスコアボード1つを表す。

保持する情報:

- board id
- board name
- created at
- updated at
- last accessed at（このボードへの最終アクセス時刻。ボード単位の自動削除判定に使う）
- game state
- team settings
- player settings
- display options
- undo history
- redo history

履歴スナップショット:

- game state、player settings、display optionsを保持する。
- 戻る、進むでは、選手名・選手成績・ピッチャー一覧・ABS表示/対戦選手表示オプションもスナップショット時点へ復元する。
- board nameとteam settingsは履歴に含めない。

表示オプション:

- ABS表示は `display options.showAbs` に保持する。
- 対戦選手表示は `display options.showMatchup` にのみ保持し、player settingsには重複して保持しない。

スコアボード名:

- スコアボード名は作成後に変更できる。
- 名称変更は試合状態やチーム設定には影響しない。
- Viewer Page、Control List Page、Score Input Pageで表示するラベルとして使う。

削除時:

- 手動削除では必ず確認を取る。
- 24時間アクセスなしの自動削除では確認を取らない。
- 削除時は、そのスコアボードの試合状態、選手名、操作履歴、専用表示オプションを削除する。
- チームプリセット、共通設定、閲覧画面設定は削除しない。

## 2. Game State

試合中に変化する情報。

保持する情報:

- inning number
- inning half: top or bottom
- balls
- strikes
- outs
- runner on first
- runner on second
- runner on third
- away score
- home score
- away ABS count
- home ABS count
- temporary overlay message
- half-inning transition display（攻守交代の中間表示。ラベルと表示終了時刻を持つ一時的な状態）
- final result（勝敗確定情報。試合終了操作で設定する永続的な状態）

制約:

- ボールは最大4表示。
- ストライクは最大3表示。
- アウトは最大3 outs表示。
- チェンジはアウト3のときのみ使用可能。
- チェンジ時は、ランナー、ボール、ストライク、アウトをリセットする。
- ランナーは一塁、二塁、三塁ごとにON/OFFできる。
- 得点は打席結果による自動加点に加えて、先攻、後攻それぞれ手動で +1 / -1 できる。
- 得点は0未満にしない。
- half-inning transition displayは、チェンジ操作のたびに新しく設定し、表示終了時刻（設定から10秒後）を持つ。表示終了時刻を過ぎたら通常表示へ戻す。表示秒数は一時演出（HOME RUN/K）の設定秒数とは独立した固定値（10秒）とする。
- final resultは、試合終了操作で設定するまでは存在しない。一度設定すると、スコアリセットの実行、または「戻る」で試合終了操作を取り消すまで保持し続ける（一時演出のような時間経過での自動消滅はしない）。試合終了操作は戻る/進むの対象であるため、game stateのスナップショットにfinal resultも含める（`operation.md` 12章参照）。
- final resultのwinnerは、試合終了操作を実行した時点の得点を比較して自動的に決める（先攻/後攻を手動で選ぶボタンは持たない）。得点の多いほうをwinnerとし、同点の場合はwinnerを持たない（どちらの帯も配色を変更しない）。

## 3. Team Settings

スコアボードごとのチーム設定。

保持する情報:

- team side: away or home
- team name
- abbreviation
- logo path
- team color
- text color
- abbreviation scale
- abbreviation width
- abbreviation centered
- linked preset id

`linked preset id` は、どのプリセットを最後に読み込んだかを記録するためだけの値であり、プリセットの変更をチーム設定へ自動反映しない。該当プリセットが削除された場合は空にする（4章）。

表示ルール:

- チームカラー上の文字色はチームごとに設定可能。
- チームカラー上の文字色の初期値は白。
- チーム略称の表示倍率はチームごとに設定可能で、初期値は100%。
- チーム略称の横幅圧縮率はチームごとに設定可能で、初期値は100%、設定範囲は30%から120%。
- チーム略称はチームごとに略称枠内の左右中央揃えを設定可能で、初期値はOFF。
- スコアボード上の得点は最も大きく表示する。

## 4. Team Preset

設定ページで保存する共通プリセット。
アプリ終了後も保存する。
スコアボード削除や24時間アクセスなしの自動削除では削除しない。

保持する情報:

- preset id
- preset name
- team name
- abbreviation
- logo path
- team color
- text color
- abbreviation width
- created at
- updated at

並び順:

- チームプリセットの表示順は、保存されているプリセット配列の順番を正とする。
- Settings Pageのプリセット一覧、プリセット並べ替え画面、Score Input Pageの編集メニューのプリセット選択肢は、この順番で表示する。
- プリセット並べ替え画面で変更した順番はサーバーに保存し、アプリ再起動後も維持する。

操作ルール:

- Settings Pageでプリセットを作成、編集、削除できる。
- プリセットを削除した場合、そのIDを `linked preset id` として持つ全スコアボードのチーム設定からIDを解除する。読み込み済みのチーム名、略称、ロゴ、色などの値は変更しない。
- Settings Pageからプリセット並べ替え画面を開き、ドラッグ操作で表示順を変更できる。
- Score Input Pageの編集メニューから、先攻/後攻それぞれへプリセットを読み込める。
- Score Input Pageの編集メニューから、現在入力中のチーム設定を新しいプリセットとして保存できる。

チームロゴのルール:

- 推奨サイズは 256x256。
- 対応形式は PNG と JPEG。
- 透過PNGは透過を保持する。
- 256x256 以外の画像は、保存時に 256x256 の正方形へ自動変換する。
- 変換時は上下幅と左右幅が等しくなるようにリサイズし、縦横比の歪みは許容する。
- 画像が大きすぎる場合も、保存時に自動圧縮する。
- 編集メニューで選択した画像は、ブラウザ側で256x256へ変換してからサーバーへアップロードする。
- サーバーは PNG/JPEG の形式を確認し、`storage/uploads/team-logos/` に保存する。
- チーム設定には、保存済みロゴへのパスを `logo path` として保持する。
- アップロード済みロゴファイルは、スコアボードやプリセットを削除しても自動では削除しない。
- Settings Pageの「スコアボード/プリセットで使われていないロゴを削除」操作で、どのスコアボード/プリセットからも参照されていないロゴファイルだけを削除できる。
- スコアボードまたはプリセットから参照されているロゴファイルは、未使用ロゴ削除の対象外にする。

## 5. Player Settings

打者・ピッチャーと試合中の選手成績を保持する情報。

保持する情報:

- away batting order 1-9
- home batting order 1-9
- away pitchers
- home pitchers
- current batting order index
- pinch hitter flag

バッター情報:

- batting order number
- player name
- is pinch hitter
- home runs
- hits
- strikeouts swinging
- strikeouts looking
- outs
- others

バッター成績の表示:

- hit count = home runs + hits
- at bat count = home runs + hits + strikeouts swinging + strikeouts looking + outs
- `others` は四死球、打撃妨害などを想定し、打数には含めない。
- 選手名メニューの打者行には、守備位置の入力欄は置かない。代わりに、上記の式で計算した「打数」「安打数」を読み取り専用の欄として表示する（編集はできない。実際のスタッツはホームラン/ヒット/三振/凡退などの打席結果ボタンでのみ変化する）。

ピッチャー情報:

- pitcher id（追加時に発行する永続ID）
- pitcher name
- pitch count
- strikeouts（空振り三振・見逃し三振の合計）
- order in pitching list

現在のピッチャー:

- 現在のピッチャーは、各チームのピッチャー一覧の最後の人とする。
- 選手名メニューからピッチャーを追加できる。
- ピッチャー名、球数、三振数は、ピッチャー一覧の行ごとに保存する。
- ピッチャーの編集・削除は配列内の位置ではなくpitcher idを対象にする。別端末が直前に別のピッチャーを追加しても、指定したピッチャー以外を変更・削除しない。
- 三振数は、打席結果が空振り三振または見逃し三振になるたびに、守備側の現在のピッチャーへ1加算する（バッター成績の`strikeouts swinging`/`strikeouts looking`とは別の、ピッチャー側の集計値）。
- 三振演出（一時演出参照）で表示する数値は、その打席で三振を奪った現在のピッチャーの三振数とする。
- 三振数は、球数と同様に選手名メニューで手動修正できる。
- 三振数は、スコアリセットで0に戻す対象に含める。

表示ルール:

- バッターは常に上。
- ピッチャーは常に下。
- 表裏に応じて色は攻撃側、守備側に入れ替わる。
- 代打時は打順番号を `PH` と表示する。

## 6. Viewer Settings

閲覧画面の表示設定。

保持する情報:

- background color
- default board scale
- board scales
- board positions
- board stack order based on last viewer adjustment
- hidden board ids

端末ごとの表示プロパティ:

- スコアボードごとの拡大率と表示位置は端末ごとに保存する。
- スコアボードサイズは拡大率から都度算出し、独立したサイズ値としては保存しない。
- スコアボードの表示位置は、ボードの右下を基準点として保存する（左上基準ではない）。攻守交代の中間表示や試合終了時に上部の対戦選手表示欄が折りたたまれてボードの高さが縮んでも、右下の位置は変わらず、上端側だけが伸縮して見えるようにするため。
- Viewer Pageで最後に位置、拡大率、表示/非表示を調整した順番を端末ごとに保存する。
- スコアボードが重なった場合は、最後に調整したスコアボードを最前面に表示する。
- 非表示にしたスコアボードIDを端末ごとに保存する。非表示は試合状態や他端末のViewerへ影響しない。
- 端末ごとの表示プロパティは、ブラウザ内の保存領域だけに保存する。
- 表示プロパティはブラウザ側で入出力できるようにする。
- 入出力により、別ブラウザや別端末へ同じ表示設定を移せるようにする。
- 他端末の表示サイズには影響させない。

入出力対象:

- スコアボードごとの拡大率
- 表示位置
- 最後に調整した順番
- 非表示にしたスコアボードID
- 背景色

## 7. General Settings

アプリ全体の設定。

保持する情報:

- auto cleanup enabled
- auto cleanup idle hours
- overlay display seconds

自動削除の初期設定:

- 稼働中スコアボードはアプリ再起動後も残す。
- ただし、各スコアボードが24時間アクセスされなかった場合は、そのスコアボードだけを削除する。ほかの稼働中スコアボードは残す（アプリ全体の最終アクセス時刻による全ボード一括削除はしない）。
- この24時間の値と有効/無効は、全体の設定メニューで管理できるようにする。
- チームプリセット、共通設定、閲覧画面設定は自動削除の対象外にする。
- 各ボードの last accessed at は、そのボードの個別取得または操作のたびに更新する（1章）。ページの表示やSSE接続だけでは更新しない。

一時演出:

- 一時演出の表示秒数は、全体の設定メニューで管理する。
- スコアボードごとの設定にはしない。

## 8. Persistence Summary

サーバーに保存するもの（アプリ終了後も残す）:

- 稼働中スコアボード（ボードごとの last accessed at を含む）
- 削除していないスコアボードの試合状態
- チームプリセット
- チームプリセットの並び順
- 共通設定

保存処理:

- 状態変更時だけ保存し、状態を変更しない一覧取得や設定取得では書き込まない。ボード個別取得は `last accessed at` が変わるため保存する。
- 同時リクエストの保存はキューで直列化し、呼び出し時点のスナップショットを受信順に書き込む。古い一時ファイルのrenameが最新状態を上書きしないようにする。
- undo historyとredo historyは`app.json`へ保存するが、ブラウザは参照しないためAPI応答とSSEの公開データから除外する。

各ブラウザ（端末）内にだけ保存するもの:

- 閲覧画面の背景色
- 閲覧画面のスコアボードごとの拡大率
- スコアボードの表示位置
- スコアボードを最後に調整した順番
- 非表示にしたスコアボードID
- 端末ごとの表示プロパティの入出力データ

閲覧画面の表示プロパティ（背景色・スコアボードごとの拡大率・表示位置・最後に調整した順番・非表示にしたスコアボードID）は、サーバーではなく各ブラウザ内にだけ保存する。
他端末の表示には影響させず、入出力機能で別端末へ移せるようにする。

スコアボード削除時に削除するもの:

- 試合状態
- 選手名
- ピッチャー一覧
- 打者成績
- 操作履歴
- そのスコアボード専用の表示オプション
