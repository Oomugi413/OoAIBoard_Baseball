# Button List

このファイルは、画面ごとに必要なボタン、メニュー、操作項目を一覧化する。
各ボタンを押したときの詳細な状態変化は `operation.md` を参照する。

## 1. Home Page

トップページに置くボタン:

- スコアボードを見る
- スコアボードを動かす

## 2. Viewer Page

スコアボードを見るページに置く操作:

- 背景色変更
- スコアボードごとの拡大率変更
- 拡大率の数値入力
- サイズの数値入力
- スコアボード表示位置変更
- スコアボード中央ドラッグによる表示位置変更
- スコアボード端ドラッグによる拡大率変更
- 最後に調整したスコアボードの前面表示
- 表示プロパティ書き出し
- 表示プロパティ読み込み

注意:

- Viewer Pageは配信利用を想定するため、最終的には操作UIを隠せる余地を残す。
- 端末ごとの表示プロパティは、ブラウザ内保存だけにする。

## 3. Control List Page

スコアボードを動かすページに置くボタン:

- 新規スコアボード作成
- スコアボード選択
- スコアボード名変更
- スコアボード削除
- 設定画面を開く

表示内容:

- スコアボードの視覚プレビューは表示しない。
- 各スコアボードに、現在の対戦スコアを `Team A 1-0 Team B 1回裏` のようなテキストで表示する。

削除時:

- 手動削除では必ず確認ダイアログを表示する。
- 24時間アクセスなしの自動削除では確認しない。

スコア入力画面に置くボタン。

- ストライク

- ボールカウント +1
- ストライクカウント -1
- ストライクカウント +1

### 4.2 Plate Appearance Buttons

- ホームラン
- ヒット
- 凡退
- 空振り三振
- 見逃し三振
- その他

### 4.3 Out Buttons

- アウトカウント -1
- アウトカウント +1
- 走塁死
- 盗塁死

### 4.4 Inning Button

- チェンジ

### 4.5 Runner Buttons

- 一塁 ON/OFF
- 二塁 ON/OFF
- 三塁 ON/OFF

### 4.6 Manual Score Buttons

- 先攻 得点 -1
- 先攻 得点 +1
- 後攻 得点 -1
- 後攻 得点 +1

### 4.7 ABS Buttons

ABS表示オプションが有効な場合に使う。

- 先攻 ABSカウント -1
- 先攻 ABSカウント +1
- 後攻 ABSカウント -1
- 後攻 ABSカウント +1

### 4.8 History Buttons

- 戻る（キーボード: Ctrl+Z / Cmd+Z）

### 4.9 Menu Buttons

- 編集メニューを開く
- 選手名メニューを開く

選手名メニューは、対戦選手表示オプションが有効な場合に使う。

## 5. Edit Menu

スコア入力画面の編集メニューに置く項目:

- スコアボード名変更
- 先攻チームロゴ設定
- 先攻チームロゴ画像選択
- 先攻チームロゴ削除
- 先攻チーム略称設定
- 先攻チームカラー設定
- 先攻文字色設定
- 後攻チームロゴ設定
- 後攻チームロゴ画像選択
- 後攻チームロゴ削除
- 後攻チーム略称設定
- 後攻チームカラー設定
- 後攻文字色設定
- ABS表示オプション ON/OFF
- 対戦選手表示オプション ON/OFF
- チームプリセット読み込み
- チームプリセット書き出し

## 6. Player Menu

対戦選手表示オプションが有効な場合に使う。

選手名メニューに置く項目:

- 先攻打者 1-9番の名前入力
- 先攻打者 1-9番の守備位置入力
- 先攻打者 1-9番の代打チェック
- 後攻打者 1-9番の名前入力
- 後攻打者 1-9番の守備位置入力
- 後攻打者 1-9番の代打チェック
- 先攻ピッチャー名入力
- 先攻ピッチャー球数入力
- 先攻ピッチャー追加
- 後攻ピッチャー名入力
- 後攻ピッチャー球数入力
- 後攻ピッチャー追加

## 7. Settings Page

全体の設定画面に置く項目:

- チームプリセット作成
- チームプリセット編集
- チームプリセット削除
- 自動削除 ON/OFF
- 自動削除までのアクセスなし時間設定
- 一時演出の表示秒数設定

## 8. Delete Confirmation Dialog

スコアボード削除時に置くボタン:

- 削除する
- キャンセル

表示する内容:

- 削除対象のスコアボード名
- 削除すると試合状態、選手名、操作履歴が消えること

## 9. Notes

- ボタン名は実装時に理解できる範囲で省略してよい。
- 例: ホームランは `HR` と表示してもよい。
- ただし、機能がわかりにくくなる省略や、ボタン同士の区別がつかなくなるような省略は避ける。
- ボタンの詳細な動作は `operation.md` を正とする。
