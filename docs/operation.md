# Operation Rules

## 1. About This File

このファイルは、実際に表示されるスコアボードやHTMLページについて、内部で実装したい機能を記したものである。
各ページのボタン操作で状態がどう変わるかを定義する。

## 2. Overview

実装対象は、配信に重ねるスコアボード本体と、それを操作・表示する5つのHTMLページ（Home / Viewer / Control List / Score Input / Settings）である。
サーバーが試合状態を保持し、操作画面からの変更をSSEで各画面へ配信する。

## 3. Pitch Buttons

ボール:

- ボールカウントを増やす。
- 対戦選手表示オプションが有効な場合、ピッチャーの球数を増やす。

ストライク:

- ストライクカウントを増やす。
- 対戦選手表示オプションが有効な場合、ピッチャーの球数を増やす。

ファウル:

- 対戦選手表示オプションが有効な場合、ピッチャーの球数を増やす。
- ストライクカウントが0または1の場合、ストライクカウントを1増やす。
- ストライクカウントが2以上の場合、ストライクカウントは増やさない。

カウント補正:

- ボールカウント -1
- ボールカウント +1
- ストライクカウント -1
- ストライクカウント +1
- カウントRS

カウント補正のルール:

- カウント補正ではピッチャーの球数を増やさない。
- カウントは下限0を下回らないようにする。
- ボールは最大4表示。
- ストライクは最大3表示。
- カウントRSはボール、ストライクのみを0へ戻し、得点、ランナー、アウト、打者、投球数などは変更しない。

補足（手動カウント・アウト操作の意図）:

- ボール、ストライク、アウトを個別に +1 / -1 できるのは、打席結果ボタンの自動処理に任せられない特殊なプレーを、手動で正しい表示に直すためである。
- 例: 四球で一塁ランナーが二塁へ飛び出してアウトになる、三振の振り逃げで打者が出塁する、3アウト後にチェンジを押す、など。
- このため、ボール4表示やアウトの手動加算といった通常と異なる状態も、一時的に作れるようにしておく。

## 4. Plate Appearance Buttons

打席結果ボタン共通:

- ホームラン、ヒット、四球、死球、凡退、空振り三振、見逃し三振、その他は打席結果ボタンとして扱う。
- 打席結果ボタンを押した場合、対戦選手表示オプションが有効ならピッチャーの球数を1増やす。
- 打席結果ボタンを押した場合、次の打者へ必ず進める。
- 打席結果ボタンを押した場合、ボール、ストライクをリセットする。
- 走塁死、盗塁死は打席結果ボタンではないため、ピッチャーの球数を増やさず、通常は次の打者へ進めない。

ホームラン:

- `HOME RUN` を一定時間表示する。
- ランナー数 + 1 を攻撃側得点に加算する。
- ランナーをリセットする。
- ボール、ストライクをリセットする。
- 成績にホームランを加算する。
- 次の打者へ進める。

ヒット:

- 成績にヒットを加算する。
- ボール、ストライクをリセットする。
- 次の打者へ進める。
- ランナーの進塁は自動では行わず、各塁のON/OFFボタンで手動調整する。

四球:

- 成績にその他を加算する。
- 打数には含めない。
- ボール、ストライクをリセットする。
- 次の打者へ進める。
- 打者を一塁へ置き、一塁走者がいる場合は野球のルールに沿って押し出しで進塁させる。
- 満塁の場合は攻撃側得点を1増やし、満塁状態を維持する。

死球:

- 四球と同じく、成績にその他を加算し、打者を一塁へ置き、押し出しでランナーを進める。
- 打数には含めない。
- ボール、ストライクをリセットする。
- 次の打者へ進める。

凡退:

- 成績に凡退を加算する。
- ボール、ストライクをリセットする。
- 次の打者へ進める。
- アウトは自動では増やさず、必要に応じてアウト操作で調整する。

空振り三振:

- `K` を3秒表示する。
- 成績に空振り三振を加算する。
- ボール、ストライクをリセットする。
- 次の打者へ進める。
- アウトは自動では増やさない。

見逃し三振:

- 逆向きの `K` を3秒表示する。
- 成績に見逃し三振を加算する。
- ボール、ストライクをリセットする。
- 次の打者へ進める。
- アウトは自動では増やさない。

その他:

- 打撃妨害など、四球・死球以外の打数に含めない結果を想定する。
- 成績にその他を加算する。
- 打数には含めない。
- ボール、ストライクをリセットする。
- 次の打者へ進める。

## 5. Out Buttons

アウトカウント -1:

- アウトを1減らす。
- アウトは0未満にしない。

アウトカウント +1:

- アウトを1増やす。
- アウトは最大3 outs表示。

走塁死:

- 現時点ではアウトカウント +1 と同じ扱い。
- 打席結果ボタンではないため、ピッチャーの球数は増やさない。
- 通常は次の打者へ進めない。

盗塁死:

- アウトを1増やす。
- 打席結果ボタンではないため、ピッチャーの球数は増やさない。
- イニング終了時も現在のバッターを維持する。

## 6. Change Button

チェンジ:

- アウトが3のときのみ使用可能。
- 表から裏、裏から次の回の表へ進める。
- ランナー、アウト、ボール、ストライクをリセットする。

例:

- 1回表から1回裏へ
- 1回裏から2回表へ

## 7. Runner Controls

ランナーON/OFF:

- 一塁、二塁、三塁をそれぞれ独立してON/OFFできる。
- ランナー操作は手動で行う。
- ヒット時の進塁は自動では行わない。
- ホームラン時は、得点加算後にすべてのランナーをOFFにする。
- チェンジ時は、すべてのランナーをOFFにする。

## 8. Manual Score Controls

得点 +1 / -1:

- 先攻、後攻それぞれの得点を手動で増減できる。
- 得点は0未満にしない。
- ホームランの自動加点とは別に、手動補正として扱う。
- 手動得点操作も戻る、進むの対象にする。

## 9. ABS Controls

ABSカウント -1 / +1:

- ABS表示オプションが有効な場合に使う。
- 先攻、後攻それぞれのABSカウントを手動で増減できる。
- ABSカウントは0未満にしない。
- ABSカウントの初期値は2回分を想定する。
- ABSカウント操作は、ボール、ストライク、アウト、得点、投球数には影響しない。
- ABSカウント操作も戻る、進むの対象にする。

対象ボタン:

- 先攻 ABSカウント -1
- 先攻 ABSカウント +1
- 後攻 ABSカウント -1
- 後攻 ABSカウント +1

## 10. Board Management Controls

スコアボード名変更:

- スコアボード名を任意の名前へ変更できる。
- 名称変更はViewer PageとControl List Pageへリアルタイム反映する。
- 名称変更は試合状態には影響しない。
- 名称変更は Score Input Page 以外（Control List Page など）から行うため、戻る、進むの対象外とする。

スコアボード削除:

- 手動削除では必ず確認ダイアログを出す。
- 削除対象のスコアボード名を確認ダイアログに表示する。
- 確認後に削除した場合、そのスコアボードの試合状態、選手名、操作履歴を削除する。
- 24時間アクセスなしの自動削除では確認を取らない。

編集メニューのチームロゴ:

- 編集メニューはMUI Drawerによるオーバーレイとして表示し、閉じるボタンを置く。
- パソコンでは操作ボタンの上に重ねて表示する。
- 先攻、後攻それぞれに PNG または JPEG のロゴ画像を選択できる。
- 選択した画像は保存前にブラウザ側で 256x256 の正方形へ変換する。
- PNG の透過は保持する。
- JPEG は白背景で 256x256 へ変換し、圧縮して保存する。
- 保存時にサーバーへアップロードし、チーム設定には保存済みロゴのパスを保持する。
- ロゴ削除を押した場合、対象チームのロゴパスを空にする。
- ロゴ削除ではロゴファイル本体は削除しない。
- Settings Pageの未使用ロゴ削除操作で、スコアボード/プリセットのどちらからも参照されていないロゴファイルだけを削除する。

編集メニューのチームプリセット:

- 先攻、後攻それぞれにチームプリセットを読み込める。
- プリセット読み込みでは、チーム名、略称、ロゴパス、チーム色、文字色、略称圧縮を入力欄へ反映する。
- 現在入力中のチーム設定を、新しいチームプリセットとして保存できる。
- プリセット保存後も、編集中のフォーム内容を勝手にリセットしない。
- プリセット読み込みやプリセット保存は、試合状態や操作履歴には影響しない。

Settings Pageのチームプリセット:

- Settings Pageでは、チームプリセットを作成、編集、削除できる。
- プリセット編集では、プリセット名、チーム名、略称、チーム色、文字色、ロゴ画像を編集できる。
- ロゴ画像は保存前にブラウザ側で 256x256 の正方形へ変換し、保存時にサーバーへアップロードする。
- プリセットカードの入力内容は、保存ボタンを押すまでサーバーへ送らない。
- プリセット編集の保存後も、他のプリセットカードの入力内容を勝手にリセットしない。
- プリセット削除では確認ダイアログを出す。削除しても、スコアボードへ読み込み済みのチーム設定は消えない。
- 未使用ロゴ削除では確認ダイアログを出し、スコアボード/プリセットのどちらからも参照されていないロゴファイルだけを削除する。

Preset Reorder Page:

- プリセット並べ替え画面では、ドラッグ用のつまみを使ってプリセットの順番を変更する。
- ドラッグ用のつまみは、仕切り線や枠を付けない横三本線アイコンにする。
- ドラッグ中のカードは浮き上がって見えるようにし、カーソル/指の位置に基づいて移動先を判定する。
- 並べ替え中は、移動先に合わせて周囲のカードが滑らかに退くアニメーションを表示し、スマホのリスト並べ替えに近い操作感にする。
- ドラッグ終了時に、変更後のプリセット順をサーバーへ保存する。
- 保存された順番は、Settings Pageと編集メニューのプリセット一覧に反映する。

編集メニューの略称拡大率:

- 先攻、後攻それぞれにチーム略称の拡大率を設定できる。
- 初期値は100%。
- 設定範囲は60%から180%までとする。
- スライダー操作は即時に反映する。
- 数値入力は入力中の値を保持し、隣の「反映」ボタンまたはEnterで確定する。
- 数値入力で範囲外の値が入った場合は、確定時に60%から180%の範囲へ丸める。
- スコアボード上のチーム略称表示に反映する。

編集メニューの略称圧縮:

- 先攻、後攻それぞれにチーム略称の横幅圧縮率を設定できる。
- 初期値は100%。
- 設定範囲は30%から120%までとする。
- 縦方向のフォントサイズは略称拡大率で決め、略称圧縮では横幅だけを変える。
- 50%の場合、縦はそのまま、横幅のみ半分にする。

## 11. Viewer Property Controls

端末ごとの表示サイズ:

- Viewer Pageでスコアボードごとに拡大率を変更できる。
- サイズ調整バーは置かず、拡大率バーに統一する。
- 拡大率の数値入力とサイズの数値入力のどちらからでもサイズを変更できる。
- 拡大率バーは操作中に即時反映し、バー下の拡大率表示も実際の表示サイズに連動する。
- 拡大率、サイズ、位置の数値入力は、隣の「反映」ボタンまたはEnterで確定する。
- 拡大率とサイズの数値入力は、入力中も互いの換算値を連動表示する。
- スコアボードの端または角をドラッグすると、OSのウィンドウサイズ変更のように拡大率を変更できる。
- スコアボードの中央部分をドラッグすると表示位置を変更できる。
- 拡大率、サイズ、表示位置の入力欄は、ドラッグ操作や端リサイズで値が変わった場合にも連動して更新する。
- Viewer Pageで位置、拡大率、サイズを調整したスコアボードは、最後に調整したものとして保存し、重なった場合に最前面へ表示する。
- 対戦選手表示オプションがOFFの場合は、対戦相手表示欄を空欄として残さず、スコアボードの縦幅を縮小して表示する。
- 変更内容はブラウザ内に保存する。
- 他端末の表示サイズには影響しない。

表示プロパティの入出力:

- ブラウザ内に保存された表示プロパティを書き出せる。
- 書き出した表示プロパティを読み込める。
- 入出力対象は、スコアボードごとの拡大率、拡大率から算出したサイズ、表示位置、最後に調整した順番、背景色を想定する。

## 12. Undo and Redo

スコアリセット:

- Score Input Pageの「履歴とメニュー」に置く。
- 実行前に確認ダイアログを表示し、ユーザーが再度了承した場合だけリセットする。
- 試合の進行状態、得点、カウント、ランナー、ABS、成績、投球数を初期状態に戻す。
- 編集メニューから設定できる項目（スコアボード名、チーム名、ロゴ、略称、色、表示オプションなど）はリセットしない。
- 選手名、守備位置、代打表示、ピッチャー一覧はリセットしない。
- リセット後は戻る/進む履歴を空にする。

戻る:

- 直前の操作を取り消す。
- Keyboard shortcut: Ctrl+Z / Cmd+Z
- キーボードショートカットは、戻る/進むボタンがある Score Input Page だけで有効にする。

進む:

- 取り消した操作をやり直す。
- Keyboard shortcut: Ctrl+Shift+Z / Cmd+Shift+Z
- キーボードショートカットは、入力欄や選択欄へ文字入力している間は反応させない。

設計方針:

- 操作ごとに game state のスナップショットを保存する。
- 戻る、進むはスコアボード単位で独立させる。
- スコアボード削除時は履歴も削除する。

履歴対象（Score Input Page での操作）:

- 投球操作
- 打席結果操作
- アウト操作
- チェンジ
- ランナーON/OFF
- 手動得点補正
- ABSカウント操作

履歴対象外（Score Input Page 以外での変更）:

- スコアボード名変更
- 選手名メニューの変更
- 閲覧画面の表示プロパティ変更
- 全体設定の変更

Score Input Page 以外から変更するものは、戻る、進むの対象にしない。

## 13. Player Menu

選手名メニュー:

- 選手名メニューはMUI Drawerによるオーバーレイとして表示し、閉じるボタンを置く。
- 先攻、後攻はタブで切り替える。
- 先攻、後攻それぞれの打順1-9番について、選手名、守備位置、代打表示を編集できる。
- 打者の選手名が変更された場合は、その打順の打撃成績をリセットする。代打(PH)フラグだけを変更した場合は成績をリセットしない。
- 先攻、後攻それぞれのピッチャー一覧を編集できる。
- ピッチャー追加ボタンで、対象チームのピッチャー行を末尾に追加する。
- 各チームのピッチャー一覧の最後の行を現在のピッチャーとして扱う。
- 投球ボタンと打席結果ボタンによる球数加算は、守備側の現在のピッチャーへ加算する。
- 選手名メニューでまとめて保存する編集は、戻る、進むの対象にしない。
- メニューを開いて入力中の内容は、保存ボタンを押すまでサーバーへ送らない。
- 複数端末で同じスコアボードを編集した場合、最後に保存した内容を有効とする。
- 選手名メニューの保存では、実際に編集した項目だけを更新する。編集していない球数、成績、他の選手情報は、メニューを開いている間に別端末で進んだ最新状態を維持する。

## 14. Realtime Sync

操作画面で何か変更したら、サーバーを経由して閲覧画面へ反映する。

基本の流れ:

1. Score Input Pageで操作する。
2. 操作内容をサーバーへ送る。
3. サーバーが状態を更新する。
4. サーバーが同じスコアボードを見ているViewer Pageへ通知する。
5. Viewer Pageが表示を更新する。

想定イベント:

- board created
- board deleted
- board updated
- board state changed
- preset changed
- active clients changed

複数端末操作:

- 複数端末から同じスコアボードを操作できるようにする。
- サーバー側の状態を正とし、各端末は操作後に最新状態を受け取る。
- 同時操作が発生した場合は、サーバーが受け取った順に処理する。
- 選手名メニューのようにまとめて保存する編集は、最後に保存した内容が有効になる（後勝ち）。
- Viewer PageとScore Input Pageの両方に、最新状態をリアルタイム配信する。
- Settings Page、編集メニュー、選手名メニューで入力中のフォーム内容は、別端末の操作によるSSE更新で勝手にリセットしない。

閲覧画面の表示プロパティ（背景色、スコアボードごとの拡大率、サイズ、表示位置、最後に調整した順番）は各ブラウザ内にだけ保存し、サーバーからは配信しない。

## 15. Automatic Cleanup

自動削除:

- 初期設定では、24時間アプリにアクセスがない場合、すべての稼働中スコアボードを削除する。
- 自動削除の有効/無効と時間は、全体の設定メニューで変更できるようにする。
- 自動削除しても、チームプリセット、共通設定、閲覧画面設定は残す。
- アプリへのアクセスは、Viewer Page、Control List Page、Score Input Page、Settings Pageの表示、またはSSE（リアルタイム）接続で更新する。
- 24時間アクセスなしの自動削除では、削除確認を表示しない。

## 16. Team Presets

Settings Page:

- チームプリセットを作成できる。
- チームプリセットのプリセット名、チーム名、略称、ロゴパス、チーム色、文字色を編集できる。
- チームプリセットを削除できる。
- Settings Pageから、プリセット並べ替え画面を開ける。
- プリセット並べ替え画面では、仕切りなしの横三本線アイコンをドラッグして順番を変更できる。
- プリセット並べ替え画面では、カーソル/指の位置に基づいて周囲のカードがアニメーションしながら退く。
- 並べ替えた順番はサーバーへ保存し、アプリ再起動後も維持する。
- Settings Pageのプリセット一覧と、Score Input Pageの編集メニューに出るプリセット選択肢は、保存済みの並び順で表示する。
- チームプリセットの作成、編集、削除はアプリ全体の共通データとして保存する。
- スコアボード/プリセットで使われていないロゴファイルを、確認後に削除できる。
- 参照中のロゴファイルは削除しない。

## 17. Overlay Timing

一時演出:

- ホームラン時は `HOME RUN` を表示する。
- 空振り三振時は `K` を表示する。
- 見逃し三振時は逆向きの `K` を表示する。
- ホームランなど通常の一時演出の表示秒数は、スコアボードごとではなく全体の設定で管理する。
- 三振演出は3秒固定で表示する。
- 三振演出の文字サイズはスコアボード縦幅の約8割にする。ホームラン演出の文字サイズは変更しない。
